B2B SaaS n8n Workflow Webflow CMS

AI Blog Automation & Webflow CMS Publishing System

Client: B2B SaaS company

Built an AI-assisted blog production workflow that turns scheduled content inputs into researched, optimized, QA-ready blog drafts and prepares them for publishing inside Webflow CMS. The workflow is intentionally detailed because it accounts for client-specific content preferences, SEO rules, formatting standards, approval steps, and publishing requirements.

Problem

The content process required multiple manual steps: topic planning, research, drafting, SEO optimization, formatting, image preparation, QA review, Webflow CMS entry, and status tracking. This made publishing slower and harder to scale consistently.

What I Built

  • Google Sheets-based schedule and configuration system for managing blog inputs
  • Content source logic supporting keywords, URLs, blog ideas, and manual overrides
  • AI research step using Perplexity Sonar Deep Research
  • AI-generated content plan, draft post, and optimized blog version
  • Webflow-ready structured output with title, slug, summary, metadata, category, and read time
  • Google Docs QA document for human review before publishing
  • AI-generated hero image flow with file upload and sharing steps
  • Webflow CMS draft creation through API integration
  • Google Sheets logging for publishing status, slug, title, image URL, and alt text

Workflow Logic

Schedule row → content source decision → AI research → content plan → draft generation → optimization → Webflow formatting → QA document → image generation → Webflow asset upload → CMS draft creation → status logging.

Human-in-the-Loop Review

The workflow creates a QA-ready Google Doc before final publishing so a human reviewer can check the content, metadata, formatting, and overall quality before it goes live.

Why the Workflow Is Detailed

The workflow includes many client-specific inputs, preferences, prompts, approval steps, SEO rules, formatting requirements, and publishing checks. The goal was not just to generate a blog post, but to preserve the client’s content standards while reducing the manual work needed to research, draft, prepare, review, and upload each article.

Construction Custom AI App Database-Backed Assistant

Internal Estimates Assistant

Client: Construction company

Built a locally hosted AI assistant concept that helps internal users ask questions about historical estimating data in plain English and receive structured, verified answers backed by database queries.

Problem

Historical estimate data was difficult to search quickly, and answering benchmark questions often required manually checking spreadsheets, past estimates, or internal project records. The team needed a faster way to access historical cost insights without relying on AI-generated guesses.

What I Built

  • Natural language question flow for internal estimating queries
  • AI routing layer to determine whether a question needed internal data or a general explanation
  • Structured query specification layer to convert user questions into controlled database requests
  • Validated SQL execution so numerical answers came from the database, not the AI model
  • Plain-English explanation layer for summarizing results for estimators and leadership
  • Backend guardrails to prevent unsupported datasets, unsafe columns, or malformed query logic

Workflow Logic

User question → AI intent routing → structured query specification → SQL validation → database lookup → verified result → plain-English explanation.

Reliability Approach

The system was designed so internal financial and estimating calculations are handled through deterministic database queries. AI is used for understanding the question and explaining results, while the database remains the source of truth.

Real Estate Education Zapier Automation Website Workflow

Real Estate Education Website Automation

Client: Real estate education company

Built and maintained a website-connected automation flow that uses site-level conditions, webhook triggers, Zapier paths, and Google Sheets updates to support course and contract-related workflows for a real estate education platform.

Problem

The site needed to handle different user actions and course-related conditions without relying on fully manual tracking. Each path required different filtering, lookup, and update logic depending on the user flow.

What I Built

  • Website-side logic and code snippets to trigger webhooks only after specific conditions were met
  • Zapier webhook intake to catch site events and route them into the correct automation path
  • Multi-path Zapier workflow for contracts and multiple course paths
  • Conditional filters to control which records continue through each path
  • Google Sheets lookup and update steps to keep course or contract tracking data current
  • Workflow troubleshooting and maintenance across the website and automation layer

Workflow Logic

Website condition met → webhook fires → Zapier catches request → path routing → conditional filtering → Google Sheets lookup → matching row update.

Why It Matters

The automation helped connect front-end website behavior with back-end tracking logic, reducing manual updates and making course or contract workflows easier to manage.
